# 互感器评审编号生成器

> 多人协作编号分配工具 —— 基于局域网共享，零服务器，双击即用

团队成员同时工作时，编号分配冲突、重复、遗漏是常见痛点。本工具通过**文件锁 + 网络共享**机制，确保每次分配的编号连续、唯一、可追溯。

---

## 核心能力

| 特性 | 说明 |
|------|------|
| 🔢 连续编号 | 自动递增，无重复无遗漏 |
| 👥 多人并发 | 文件锁排队机制，多人同时操作互不干扰 |
| 📁 局域网共享 | 基于 SMB 共享文件夹，无需部署服务器 |
| 📋 历史追溯 | 记录每次操作的用户、时间、编号，支持搜索 |
| 📝 自动建文档 | 分配编号时自动创建对应 `.docx` 文件 |
| 🚀 绿色免安装 | 打包为单个 exe，双击即用 |

## 工作原理

```
用户点击"获取编号"
    ↓
创建 lock.tmp（原子操作，互斥锁）
    ↓
读取 current_number.txt → +1 → 写回
    ↓
写入日志 number_log.json
    ↓
删除 lock.tmp，释放锁
```

**超时保护**：锁等待上限 10 秒，过期锁（>30 秒）自动清理。

---

## 数据存储

所有数据存放在共享路径 `\\192.168.2.64\e\编号生成器数据\` 下：

| 文件 | 用途 |
|------|------|
| `current_number.txt` | 当前最大编号 |
| `number_log.json` | 操作历史（最近 1000 条） |
| `config.json` | 起始编号等配置 |
| `lock.tmp` | 临时锁文件（自动生成/删除） |

---

## 配置

- **共享路径**：在 `number_manager.py` 第 9 行修改 `DEFAULT_SHARED_PATH`
- **起始编号**：首次使用时通过界面设置，或直接编辑 `config.json`

## 常见问题

**Q: 提示"无法访问共享路径"**
检查网络连接和共享文件夹权限，确保所有用户都能读写该路径。

**Q: 多人同时使用时会卡吗？**
程序自动排队，单次操作通常 0.1 秒内完成，体感无延迟。

**Q: 编号会重复吗？**
文件锁保证原子操作，正常使用不会重复。如遇网络异常请重启程序。
